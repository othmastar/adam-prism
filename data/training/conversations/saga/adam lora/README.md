# 🤖 آدم - LoRA Fine-Tuning Pipeline
# تدريب موديل AI مخصص على بياناتك الشخصية

## ═══════════════════════════════════════════════════════════════
## 🎯 إيه اللي مختلف عن قبل؟
## ═══════════════════════════════════════════════════════════════

### المشكلة القديمة
كنا بنحاول نخلي Gemma 4B يتصرف كـ "آدم" عن طريق **كلمات** (Prompt Engineering).
النتيجة: الموديل كان ذكي لوحده بس "بيبقى غبي" جوه البرنامج.

**السبب:** مفيش موديل بيتغير سلوكه بالكلمات لوحدها - لازم يتدرب.

### الحل الجديد
**LoRA Fine-Tuning** = تدريب حقيقي بيغيّر أوزان الموديل فعلاً.

الفرق زي:
- ❌ القديم: تحاول تخلي واحد يتكلم لغة بطريقة الإيحاء
- ✅ الجديد: تعلّمه اللغة فعلاً

## ═══════════════════════════════════════════════════════════════
## 🧠 إيه LoRA؟
## ═══════════════════════════════════════════════════════════════

**LoRA** = Low-Rank Adaptation

تقنية اتعملت سنة 2021 وبقت المعيار لتدريب الموديلات الكبيرة.

**الفكرة:**
- الموديل الأصلي: 4 مليار باراميتر (أوزان)
- LoRA بيضيف: ~4 مليون باراميتر بس (0.1%)
- التأثير: الموديل بيغيّر سلوكه فعلاً

**ليه ده مش تقنية قديمة؟**
- DeepSeek و Google و Meta كلهم بيستخدموا LoRA أو تقنيات مشابهة
- الفرق: هم عندهم بيانات أكتر وكومبيوتر أكتر
- بس **التقنية نفسها هي هي**

## 📊 البيانات - أهم جزء!
## ═══════════════════════════════════════════════════════════════

### الكمية
| الكمية | النتيجة |
|--------|---------|
| 10-30 مثال | سلوك أساسي بس |
| 50-100 مثال | سلوك واضح (يُنصح للبداية) |
| 500-1000 مثال | نتائج قوية |
| 2000+ مثال | نتائج احترافية |

### الصيغة (ShareGPT)
```json
{
  "conversations": [
    {"role": "system", "content": "أنت آدم، المساعد الشخصي لأسامة..."},
    {"role": "user", "content": "السؤال"},
    {"role": "assistant", "content": "الجواب"}
  ]
}
```

### إزاي تجمع بيانات؟

1. **يدوياً (الأدق):** اكتب محادثات تمثل الاستخدام الحقيقي
2. **من ChatGPT/Claude:** اطلب منه يولّد أمثلة
3. **من محادثاتك:** جمّع أفضل محادثاتك مع AI
4. **من مستنداتك:** حوّل تقاريرك وملاحظاتك لـ Q&A

### نصائح للبيانات
- **التنوع أهم من الكمية**: 100 مثال متنوع أحسن من 1000 مثال متشابه
- **اكتب الردود بنفسك**: لو عايز آدم يتكلم زيّك، اكتب الردود بأسلوبك
- **أضف أمثلة سلبية**: مش بس الصح - اكتب إزاي آدم يرد على أسئلة غلط
- **اختبر البيانات**: `python3 prepare_data.py --validate data/training_data.jsonl`

## 🔍 إزاي تعرف إن التدريب نجح؟
## ═══════════════════════════════════════════════════════════════

1. **الـ Loss**: لازم ينزل مع الوقت
   - يبدأ من ~2.5
   - ينزل لـ ~1.0-1.5 = كويس
   - ينزل لـ ~0.5 = ممتاز
   - يوصل لـ ~0.0 = overfitting (مش كويس!)

2. **الردود**: جرّب الموديل وشوف
   - لو بيرد بشكل منطقي = نجاح
   - لو بيرد بكلام غريب = محتاج بيانات أكتر
   - لو بيرد بالظبط زي بيانات التدريب = overfitting

3. **التكرار**: جرّب نفس السؤال أكتر من مرة
   - لو الردود متنوعة بس كلها صحيحة = ممتاز
   - لو دايماً نفس الرد = محتاج تنوع في البيانات

## 💬 مقارنة: قبل وبعد LoRA
## ═══════════════════════════════════════════════════════════════

| | قبل (Prompt Engineering) | بعد (LoRA) |
|---|---|---|
| التغيير | كلمات في الـ prompt | أوزان الموديل فعلاً |
| السلوك | بيتصنع | طبيعي |
| الذاكرة | محدودة بالـ context | مدمجة في الموديل |
| الاستقرار | يتغير حسب الـ prompt | ثابت |
| المعرفة | عامة | متخصصة |
| السرعة | نفس السرعة | نفس السرعة |

## 🛠️ استكشاف الأخطاء
## ═══════════════════════════════════════════════════════════════

### OOM (Out of Memory)
```
قلّل: per_device_train_batch_size = 1
قلّل: max_seq_length = 1024
قلّل: lora_r = 8
أو: استخدم Colab بموديل أصغر
```

### الردود غريبة بعد التدريب
```
قلّل: max_steps (overfitting)
قلّل: learning_rate = 1e-4
زوّد: تنوع البيانات
```

### Loss مش بينزل
```
زوّد: learning_rate = 3e-4
تأكد: البيانات صحيحة (validate)
جرّب: موديل أساسي مختلف
```

### Unsloth مش يشتغل
```
استخدم: --method peft
أو: pip install unsloth --upgrade
```
