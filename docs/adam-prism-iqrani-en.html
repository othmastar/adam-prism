Project One

Petroleum SCADA with RAG System

First real project in a critical sector. A SCADA (Supervisory Control and Data Acquisition) system for a petroleum network with RAG (Retrieval-Augmented Generation) capabilities. This is where I understood that real systems must stay alive 24/7 — not just work on a dev machine. First lesson: if your system crashes, the problem isn't the code — it's the architecture.

Critical Sector 24/7 Uptime RAG System
Project Two

Pharmaceutical Exchange — 12,000 Messages Daily

A system handling 12,000 messages per day from WhatsApp supply and demand groups, plus 40 Excel sheets each containing 400 items with daily price updates. The client paid zero to Meta — I built them a complete system that processes all of this automatically. I used SQL cache to filter duplicates, Redis to manage input flow, then routed everything to persistent PostgreSQL, and connected it all to Metabase dashboards. Then I put an LLM in control — it collects client data and sends alerts when an item they want to buy or sell becomes available or gets requested. 10 real KPIs running live.

12K msgs/day 40 sheets x 400 items Redis + PostgreSQL LLM-controlled 10 KPIs $0 to Meta
Project Three

Digital Twin for Petroleum Pipeline Company

Same scale — but bigger. A system handling up to 100,000 entries daily across a scale of 300 valve rooms. This is where I learned that performance isn't a luxury — it's the difference between a running system and a dying one. When your system receives 100,000 requests per day, either the architecture holds or it dies. And that taught me that every component must be independently isolatable — so when something stops working, the rest keeps running.

100K entries/day 300 valve rooms Digital Twin Fault Isolation
Milestone Four

The Shift to Linux — Sovereignty Starts at the OS

After seeing what happens when your data lives on someone else's servers, moving to Linux was a natural decision. Digital sovereignty isn't just in the code — it's in the entire stack. When you control every layer, from kernel to application, you actually understand what's happening. And that's what made Adam Prism run 100% locally from day one of its design.

Linux Full Stack Control Digital Sovereignty
Project Five

Digital Twin of a Legal Expert

A digital lawyer expert in law, connected to vector memory and an SFT LLM, with an extraordinary interface. Here I learned what "real memory" means — not just logging conversations, but understanding context, instant retrieval, and continuity. When a client returns after a week, the system must remember who they are, what their case is, and what was discussed last time. Without vector memory, this is impossible. And that's why Adam Prism embeds memory from day one.
