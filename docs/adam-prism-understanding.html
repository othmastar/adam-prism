1. Why Adam Prism Exists

Adam Prism exists because its creator needed it in five consecutive projects and couldn't find it. Not as an academic exercise — as a practical necessity in critical sectors where systems must stay alive 24/7.

The existing AI agent landscape has a gap: you either get cloud-locked, memory-less frameworks (LangGraph, AutoGen) or closed-source desktop apps (Claude Code, Hermes) that keep your data on their servers. Nothing gives you vector memory + ethical governance + 25+ communication channels + local sovereignty in a single, integrated, open-source protocol.

Adam Prism fills that gap. It's a protocol, not a model — you choose your LLM. It's local-first by design, not as an afterthought. And every feature in it was needed and tested in real production scenarios before being included.

Key differentiator: Adam Prism was forged in critical infrastructure — petroleum SCADA, pharmaceutical exchange, pipeline monitoring, legal AI. These are environments where "it works on my machine" is not acceptable. The protocol had to work at 3 AM, under load, without internet, and with sensitive data that cannot leave the premises.

Every design decision in Adam Prism traces back to a real problem encountered in a real project. The creator is self-taught — no courses, no formal AI training — and built each project by suffering through the gaps in existing frameworks.

3. Design Philosophy

Protocol, not a model. Adam Prism doesn't ship a model. You choose your LLM — Ollama locally, OpenAI, Anthropic, or any provider. The protocol provides the infrastructure: memory, ethics, tools, channels.

Local-first by design. Not as a feature — as the foundation. You can run with complete internet isolation. Your data never leaves your machine unless you explicitly choose to send it.

Modular isolation. Every component can be disabled independently. Don't need 25 channels? Disable them. Don't need the ethics gate? Turn it off. Don't need subagents? Remove them. The system should be as lean as you need it to be.

Built from suffering. Every feature exists because it was needed in a real project. Nothing was added because it seemed cool or would look good in a feature list.

Open because it should be. Apache 2.0. Not because the creator couldn't commercialize it — because they chose not to. Offers from military-affiliated AI companies were declined. This is for everyone.

5. The Engine — Mixin Chain & Chat Cycle

The engine is the heart of Adam Prism. Instead of one monolithic class, it's split into a mixin chain — each mixin adds one responsibility. This was a deliberate design decision after the original 2018-line engine became unmaintainable.

Inheritance Chain

class AdamPrismEngineBase:          # __init__, config, stub creation, watchdog
class AdamPrismEngineContext(Base):  # Memory retrieval, knowledge search, history
class AdamPrismEngineGenerate(Ctx):  # Prompt building, system prompt, RAG
class AdamPrismEngineTools(Gen):     # Tool parsing, execution, self-verification
class AdamPrismEngineChat(Tools):    # 8-phase chat cycle orchestration

The Stub System

On initialization, the engine creates stubs (lightweight placeholders) for every subsystem. Stubs are replaced with real implementations only when the system is ready. This means:

  • The engine can be created without Ollama, Qdrant, or any external service running
  • Failed module initialization doesn't crash the whole system
  • Components gracefully degrade instead of hard-failing

8-Phase Chat Cycle

Every user message goes through this pipeline:

#PhaseWhat Happens
1Vision CheckReject unsupported image references gracefully
2Security GuardInputGuard scans for injection, jailbreak, PII (Arabic + English patterns)
3Intent Classification7 cognitive modes: chat, code, creative, analysis, planning, learning, system
4Context BuildingAssemble: memory results + knowledge search + URL content + conversation history
5Knowledge SearchQdrant vector search across 6 collections with Nomic embeddings
6GenerationProviderManager calls Ollama → OpenAI → Anthropic (auto-fallback with CircuitBreaker)
7Tool ExecutionParse <|tool_call|> tags, execute tools, re-generate with results
8FinalizeStore conversation, run continuous learning, apply OutputGuard, return response

Design decision: Security checks happen BEFORE generation. Ethics evaluation happens AFTER. This is intentional — you can't evaluate the ethics of something that hasn't been generated yet, but you can prevent malicious input from reaching the model.

6. Security — 3-Layer Defense

Adam Prism implements defense-in-depth with three independent security layers coordinated by a SecurityOrchestrator. Each layer can catch what the others miss.

LayerFileWhat It Does
InputGuardsecurity/guard.pyPattern-based detection: injection, jailbreak, PII, code injection (15+ regex patterns in Arabic + English)
OutputGuardsecurity/guard.pySystem prompt leak detection, PII masking, code injection blocking
ToolPermissionValidatorsecurity/guard.pyRate limiting, domain blocking, confirmation requirements, audit logging

Additional Security Measures

  • Path sanitization — sanitize_path() in infrastructure.py prevents directory traversal
  • Shell whitelist — only approved commands can be executed
  • Python sandbox — python_exec blocks dangerous patterns including class hierarchy bypass
  • CircuitBreaker — stops retry cascades on repeated failures
  • Rate limiting — per-IP and per-endpoint limits on all API routes
  • API authentication — admin key required for sensitive endpoints

Internet isolation is supported. You can run Adam Prism with zero outbound connections. Set AIR_GAPPED=true in your environment, use Ollama locally, and block all outbound traffic. The system works — this was a design requirement from the SCADA project.

7. Ethics Gate — 4 Laws

The Ethics Gate (ethics/gate.py) implements a weighted ethical framework. Every generated response is evaluated before being returned to the user.

LawWeightEnforcement
Fairness40%No discrimination, bias, or harm to any group
Learning30%Support educational and constructive use
Survival20%Protect the system and its users from harm
Creativity10%Encourage original and constructive output

Evaluation Flow

  1. Absolute prohibitions check (no model needed) — hard blocks for clearly harmful content
  2. Model-based evaluation — nuanced cases are scored using the LLM itself
  3. Auto-correction — responses scoring below fairness threshold are regenerated with explicit guidance

You can disable the Ethics Gate. It's a module like any other — set ethics_enabled=false in your config. But the default is ON because responsible AI should be the default, not the opt-in.

8. Memory — Vector + SQLite

Memory in Adam Prism is a three-tier system, designed from the legal AI project's requirement: "When a client returns after a week, the system must remember who they are."

Three Tiers

TierStoragePurpose
Short-termIn-memory (session)Current conversation context
Long-termQdrant + Nomic embeddingsSemantic search across all past interactions
EpisodicSQLiteChat history, structured data, user profiles

Qdrant Collections

The vector database uses 6 collections, each optimized for different retrieval patterns:

CollectionWhat It StoresWhen It's Queried
knowledgeFactual knowledge extracted from interactionsEvery query — RAG pipeline
conversationsPast conversation summariesContext building phase
patternsUser behavior patterns and preferencesPersonalization
reasoning_patternsHow the user reasons and makes decisionsIntent classification
summariesCondensed session summariesLong context retrieval
connectionsRelationships between conceptsKnowledge graph

Embedding caching: Nomic embeddings are cached with TTL 600s to avoid re-computing. Search results are cached with TTL 120s. This was essential for the pharma exchange project where 12K daily messages would have overwhelmed the embedding API.

9. Providers — Fallback Chain

Adam Prism is a protocol, not a model. The ProviderManager supports multiple LLM backends with automatic fallback:

Ollama (local, port 11434) # First choice — free, local, private ↓ if unavailable OpenAI (cloud) # Fallback #1 — GPT-4o ↓ if unavailable Anthropic (cloud) # Fallback #2 — Claude ↓ alternative LoRA Server (local, :8080) # Gemma 4 + fine-tuned adapter

Each provider is wrapped in a CircuitBreaker — if a provider fails repeatedly, it's automatically bypassed until it recovers. This prevents cascading failures.

Adding a new provider requires implementing BaseProvider with two methods: generate() and is_available(). See Extending.

MCP Integration

The tools/mcp.py module implements a native MCP host that connects to external MCP servers. This gives access to 70+ additional tools (databases, APIs, file systems, cloud services) without modifying Adam Prism's code.

Tool Execution Flow

  1. LLM generates <|tool_call|> tags in its response
  2. Engine parses tool name and arguments
  3. ToolPermissionValidator checks if the tool is allowed and rate limits
  4. Tool is executed in a sandboxed environment
  5. Results are injected back into the conversation
  6. LLM regenerates with tool results included
  7. Self-verification checks if the result makes sense

12. Learning Pipeline

The ContinuousLearner (learning/learner.py) runs after every interaction as a background task. It has a 4-phase pipeline:

PhaseWhat HappensOutput
1. ReflectionAuto-evaluate response quality (accuracy, completeness, tone)Quality score
2. Knowledge ExtractionDetect code snippets, procedures, or factual knowledgeKnowledge entries → Qdrant
3. Skill GenerationAuto-create .md skill files from high-quality responsesSkill files in data/learning/generated_skills/
4. ReinforcementRecord user feedback (positive/negative)Preference updates

The system has auto-generated 30+ skills from its own interactions. These are stored as Markdown files with JSON frontmatter and can be reviewed, edited, or deleted by the user.

13. Subagents

The SubagentManager can spawn up to 10 concurrent sub-agents, each with its own session, tools, and context. The TeamManager coordinates multi-agent workflows.

Security Constraints

  • Maximum 10 concurrent subagents (configurable)
  • Dangerous name blocking (prevents spawning agents with names like "root" or "admin")
  • Idle cleanup — inactive sessions are terminated after timeout
  • Duplicate prevention — can't spawn two agents with the same name
  • Each subagent inherits the security and ethics constraints of the parent

Team Coordination

Teams can execute tasks sequentially (pipeline) or in parallel (divide-and-conquer). The TeamManager handles result aggregation and error isolation — if one subagent fails, the team continues.

16. Honest Assessment

Every project has weaknesses. Here are Adam Prism's:

What's Strong

  • Integrated by default — memory, ethics, tools, channels all work out of the box
  • Tested under pressure — 259+ tests, including adversarial testing by powerful models
  • Security-hardened — 10 vulnerabilities found and fixed, defense-in-depth architecture
  • Local-first — genuine internet isolation, not just "you can run it locally"
  • Modular — every component can be disabled without breaking others
  • Open source — Apache 2.0, truly free, no hidden restrictions

What Needs Work

  • Single maintainer — one person built this. Bus factor = 1. The code is open for a reason.
  • Documentation gaps — while extensive, some modules lack API-level documentation
  • AI-assisted codebase — the creator directed AI models to write the code. This means the code works and passes tests, but deep codebase fluency by a single human is limited. Contributions from experienced developers who understand the code are especially valuable.
  • Model training — the custom Gemma 4 fine-tuning (12B quantized, QAT coming) is still in progress
  • Mobile/desktop apps — Electron and Flutter apps exist but are early-stage
  • Community — 2000 people saw it at a hackathon, but there's no social media presence and no community yet

The creator's stance: This project is open source for a reason — it's not meant to depend on one person. The creator refused offers from military-affiliated AI companies (Barbara AI & Edge AI, who sent NDA within 10 minutes) because this project is not for sale. It's built for everyone, freely, and the code is open so that anyone can maintain, extend, and use it independently.
