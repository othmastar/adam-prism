نظرة عامة على المشروع

ما هو Adam Prism؟

Adam Prism هو إطار عمل توأم رقمي مفتوح المصدر يعمل محلياً بالكامل على أجهزتك. يجمع بين بنية وعي اصطناعي مكونة من 12 طبقة، و7 أوضاع معرفية، و4 قوانين أخلاقية. يتواصل عبر 25 قناة، ويستخدم 53 أداة مدمجة + 70+ أداة MCP، ويتعلم باستمرار من كل محادثة.

اسم "بريزم" (Prism) مستوحى من المنشور الذي يحوّل الضوء الأبيض إلى ألوان الطيف — كذلك آدم يحوّل بياناتك الخام إلى رؤى وأفكار ملونة.

أوضاع معرفية

المبادئ الأساسية

السيادة عبر المحلية
كل شيء يعمل على أجهزتك. لا بيانات تخرج إلا باختيارك. لا أحد يستطيع تغيير النموذج أو رفع السعر أو إيقاف الخدمة.
الحرية عبر التفكيكية
كل قطعة يمكن إزالتها أو استبدالها أو توسيعها. تريد قناة واحدة بدون ذاكرة؟ تم. تريد 100 أداة مخصصة؟ أضفها.
التواضع بالتصميم
آدم يعترف عندما لا يعرف. يطلب المساعدة منك أو من نماذج أكبر. هذا ليس بديلاً — إنه مبدأ تصميم.
النمو عبر العلاقة
آدم يتأمل ويتعلم ويتغير مع الوقت. لا يولّد ردوداً فحسب — بل ينمو معك.

نظام Auto-Fallback

💡

المدير (ProviderManager) يحاول مع المزود الحالي أولاً، وإذا فشل ينتقل تلقائياً للمزود التالي حسب ترتيب provider_fallback.

// في config/default.json
{
  "provider_fallback": ["ollama", "openai", "anthropic"]
}

الترتيب الافتراضي: Ollama (محلي) ← OpenAI (سحابي) ← Anthropic (سحابي). يمكنك تغيير الترتيب حسب احتياجك.

نظام الذاكرة

نظام الذاكرة في Adam Prism متعدد الطبقات يجمع بين الذاكرة الشعاعية (Qdrant) والذاكرة المحلية (SQLite) مع نظام تخزين مؤقت ذكي.

الذاكرة الشعاعية — Qdrant

تخزين واسترجاع دلالي باستخدام التضمينات (Embeddings). كل نص يُحوّل إلى 768 رقم عبر نموذج nomic-embed-text ويُخزّن في Qdrant للبحث بالمعنى لا بالكلمات.

المجموعةالمحتوىمثال
adam_knowledgeمعرفة عامة"مبدأ عدم اليقين ينص على..."

adam_patternsأنماط تفكير"المستخدم يفضل الإجابات المنهجية"
adam_reasoning_patternsأنماط الاستدلال"عند التحليل، ابدأ بالفرضيات"
adam_summariesملخصاتملخص كتاب "تفكير سريع وبطيء"
adam_connectionsروابط"مبدأ عدم اليقين ↔ اتخاذ القرار"

حارس الأمان (Security Guard)

يتكون من 3 طبقات حماية تعمل بدون استدعاء النموذج (سريعة وخفيفة):

1
InputGuard — فحص المدخلات قبل وصولها للنموذج. يكشف: حقن الأوامر، تسريب تعليمات النظام، محاولات كسر القيود، حقن الكود.
2
OutputGuard — فحص المخرجات قبل وصولها للمستخدم. يكشف: تسريب system prompt، بيانات شخصية (PII)، حقن كود.
3
ToolPermissionValidator — التحقق من صلاحيات استدعاء الأدوات: حد الاستدعاءات، النطاقات المحظورة، التأكيد المطلوب.

كل استدعاء أداة يُسجّل مع: اسم الأداة، المعاملات، السماح/المنع، السبب، الوقت. يمكنك الوصول عبر:

حماية SSRF

  • فحص عناوين URL قبل طلبها لمنع الوصول للشبكة الداخلية
  • تعقيم محتوى الويب غير الموثوق عبر Base64 encoding
  • تحديد أقصى لحجم المحتوى (max_chars=4000)

أمان Shell

  • تنفيذ الأوامر في بيئة محدودة (sandbox)
  • حد أقصى 20 استدعاء shell لكل جلسة
  • مراقبة الأوامر الخطيرة (os.system, subprocess.run, exec())
  • تسجيل كل أمر في سجل التدقيق

نظام الإضافات (Plugins)

نظام إضافات مبني على الـ hooks يسمح باعتراض وتعديل سلوك المحرك في نقاط محددة:

1
before_generate() — تعديل الرسالة/السياق قبل توليد الرد
2
after_generate() — تعديل الرد بعد التوليد
3
before_tool() — اعتراض تنفيذ الأداة (أو منعها)
4
after_tool() — تعديل نتيجة الأداة
# مثال: إنشاء إضافة بسيطة
from adam.plugins import AdamPlugin

class MyPlugin(AdamPlugin):
    name = "my-plugin"
    version = "1.0.0"
    description = "Does something useful"

    async def on_load(self, engine):
        self.engine = engine

    async def before_generate(self, message, context):
        context["extra_info"] = "added by plugin"
        return {"message": message, "context": context}

    async def after_generate(self, message, response):
        return response + "\n\n— Powered by MyPlugin"

نظام الجدولة (Scheduler)

يدعم جدولة المهام بثلاثة أنواع:

Cron دوري
تنفيذ مهمة بجدول زمني محدد (مثل: كل يوم الساعة 8 صباحاً)
Interval فاصل
تنفيذ مهمة كل فترة زمنية محددة (مثل: كل 5 دقائق)
One-time مرة واحدة
تنفيذ مهمة في وقت محدد مرة واحدة فقط
# عبر API
POST /api/scheduler/cron
{
  "job_id": "daily-summary",
  "cron_expr": "0 8 * * *",
  "name": "ملخص يومي"
}

POST /api/scheduler/interval
{
  "job_id": "health-check",
  "seconds": 300,
  "name": "فحص صحي"
}

POST /api/scheduler/once
{
  "job_id": "backup-now",
  "run_at": "2026-03-05T02:00:00",
  "name": "نسخ احتياطي"
}

# إدارة المهام
GET  /api/scheduler/list        # عرض كل المهام
GET  /api/scheduler/{job_id}    # تفاصيل مهمة
DELETE /api/scheduler/{job_id}  # حذف مهمة

أمثلة على Cron

التعبيرالوصف
0 8 * * *كل يوم الساعة 8:00 صباحاً
0 */2 * * *كل ساعتين
30 0 * * 1كل اثنين الساعة 12:30 بعد منتصف الليل
0 0 1 * *أول يوم من كل شهر

صيغة Cron: دقيقة ساعة يوم شهر يوم_الأسبوع — 5 أجزاء مفصولة بمسافات.

النسخ الاحتياطي والاستعادة

نسخ البيانات احتياطياً

# 1. نسخ بيانات Qdrant
docker exec adam-qdrant curl -X POST http://localhost:6333/snapshots
# النسخة تُحفظ في /qdrant/snapshots داخل الحاوية

# أو نسخ المجلد مباشرة
cp -r qdrant_data/ backup/qdrant_$(date +%Y%m%d)/

# 2. نسخ بيانات API (محادثات وسجلات)
cp -r data/ backup/data_$(date +%Y%m%d)/
cp -r logs/ backup/logs_$(date +%Y%m%d)/

# 3. نسخ نماذج Ollama
cp -r ~/.ollama/ backup/ollama_$(date +%Y%m%d)/

# 4. نسخ شامل بسكريبت واحد
tar -czf adam-backup-$(date +%Y%m%d).tar.gz \
    qdrant_data/ \
    data/ \
    logs/ \
    config/ \
    notebook/

نسخ الإعدادات احتياطياً

# نسخ ملفات الإعداد
cp config/default.json backup/config_$(date +%Y%m%d).json
cp .env backup/env_$(date +%Y%m%d)

# نسخ مهاراتك المخصصة
cp -r ~/.adam/skills/ backup/skills_$(date +%Y%m%d)/

# نسخ إضافاتك
cp -r plugins/ backup/plugins_$(date +%Y%m%d)/

إجراءات الاستعادة

# 1. استعادة Qdrant
# أوقف الحاوية
docker stop adam-qdrant

# استعادة البيانات
cp -r backup/qdrant_YYYYMMDD/ qdrant_data/

# أعد تشغيل الحاوية
docker start adam-qdrant

# 2. استعادة API
cp -r backup/data_YYYYMMDD/ data/
cp -r backup/logs_YYYYMMDD/ logs/

# 3. استعادة الإعدادات
cp backup/config_YYYYMMDD.json config/default.json
cp backup/env_YYYYMMDD .env

# 4. استعادة من أرشيف شامل
tar -xzf adam-backup-YYYYMMDD.tar.gz

# 5. إعادة تشغيل الكل
cd deploy && docker compose restart
⚠️

أوقف كل الخدمات قبل الاستعادة لتجنب تلف البيانات. استخدم bash stop.sh أو docker compose down.
